Movie Site Design Process

About the work

The following project was designed to broaden my knowledge and experience with UX design and allow me to follow every step as a UX designer.

More about the work

Project Brief

The following project takes a look into the research and design methods with the aim to create a greater design for a movie ratings website. Existing movie ratings sites were broken down in the process of research, to gain third party, user opinionated results, to find out what aspects of individual sites were most popular. Web design literature was gathered to improve on these aspects before building them all as one single site. The project site, created at the end of the research process, was then tested to see if this method of design works well when creating a new product.

Aim

The main aim of this project is to create a well-designed and efficient site that allows for users to gain information quickly. The project must hold an efficient navigation system along with a modern and elegant design that can run on any device but is pleasant to look at.

Methdology

A methodology is basically always needed when it comes to a project. The waterfall methodology met this projects specifications and time line quite well. The limited time meant that changes couldn’t be made to the project along the way, especially since only one person would be working on the project. Apposed to an agile methodology, the waterfall method doesn’t allow changes along the projects build.

  • Requirements

    After my thorough analysis of the 10 websites chosen with the use of the surveys, user testing, ratings and product review, i created a list of requirements for the end product. These requirements included what languages to use, what elements to include and even what types of colour to use.

  • Analysis

    The requirements list is analysed to get the design stage ready. This allows for me to know every feature that needs to be included in the design(s).

  • Design

    I design a set of wireframe designs with the simple use of a pencil and paper. Once im happy with one design i move over to the PC and create a set of final wireframes in photoshop. I then move on to create the final designs of the product with the use of the wireframes as a guide... a bit like a colouring book.

  • Code

    Now is the time to create the low fidelity designs. These designs dont need to be 100% like the end product but as close to it as possible. I created a super low fidelity design here so users can get a feel for the navigation system i implemented.

  • Testing

    The testing stage followed the same guide as my research stage... mostly. The way i gained user research for the initial research is what was reused here. I could then compare results of my site to the other 10 tester sites directly and get a set of accurate results.

Research Methods

Product Review

A set of 10 popular movie review sites were chosen to be the subjects for review. These sites were all chosen based on the popularity of the sites in the category of movie review websites. To measure the popularity of each site, Similarweb and Sitechcker were used to measure the traffic that each site produced. The top 10 movie ratings sites that then showed the highest traffic were then picked.

Survey

A survey allows for users to complete or attempt to complete a set of tasks to get a greater insight in the usability of multiple sites.
