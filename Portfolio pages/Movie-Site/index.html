Project Brief

The following project takes a look into the research and design methods with the aim to create a greater design for a movie ratings website. Existing movie ratings sites were broken down in the process of research, to gain third party, user opinionated results, to find out what aspects of individual sites were most popular. Web design literature was gathered to improve on these aspects before building them all as one single site. The project site, created at the end of the research process, was then tested to see if this method of design works well when creating a new product.

Aim

The main aim of this project is to create a well-designed and efficient site that allows for users to gain information quickly. The project must hold an efficient navigation system along with a modern and elegant design that can run on any device but is pleasant to look at.

Objectives

  • A specification list must be created in order to create the basis of what the site should hold, on both information and the design of the aspects of each feature present and the design of the site looks.
  • A product review on movie review/ratings needs to be set out the gain the opinions of multiple users to look at what can be improved and even what aspects of such sites can be adopted.
  • A literature review needs to be conducted to gain web design and development and User Experience (UX) knowledge to help towards the design of the product.
  • An end design needs to be created for each main page of the site, such as the home page, to create the basis towards the development of the site.
  • A mock-up needs to be created in order to have a low-level version of the product created. This will allow for basic testing of the product and allow to gain feedback on if the new design is successful.
  • A number of users are in need to test the product created to conclude if the project is successful.

Research Methods

    Product Review

  • A product review allows the collection of people’s opinions/takes on currently available products.

    • Survey

  • A survey allows for users to complete or attempt to complete a set of tasks to get a greater insight in the usability of multiple sites.

    • Testing

  • Tests are taken alongside the surveys to measure aspects such as how long it takes for someone to navigate the site for example. This can also test the efficiency of a site.

    • Rating

  • A simple /10 rating method is also included in the research to gain a further insight into what users think of website elements.

Tools Used

Methdology

A methodology is basically always needed when it comes to a project. The waterfall methodology met this projects specifications and time line quite well. The limited time meant that changes couldn’t be made to the project along the way, especially since only one person would be working on the project. Apposed to an agile methodology, the waterfall method doesn’t allow changes along the projects build.

  • Requirements

    I broke down the brief to create a requirements list. This includes everything from the colours to use, to every piece of content that needs to be included.

  • Analysis

    The requirements list is analysed to get the design stage ready. This allows for me to know every feature that needs to be included in the design(s).

  • Design

    Usually with Photoshop, i will then create a set of draft designs. I then choose the best design and refine it.

  • Code

    I now code the product page by page based on the final design. Visual code is definetly my choice of software here!

  • Testing

    Now it's time to look for bugs! Every feature is tested and i look at every page at as low and high of a resolution as i can. Usually from 250px - 1440px.

Languages
